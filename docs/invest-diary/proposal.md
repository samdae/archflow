가제: 투자일기

1. 사용자에게 먼저 포트폴리오를 요청함.
2. 투자일기 작성 시 당일 매매기록을 기록함
사용자가 기존에 작성 해 놓은 포트폴리오를 변경하는 방식을 통해
무엇이 얼마나 추가되고 무엇이 얼마나 빠졌는지 기록함.
그것을 '매매기록'이라고 부르겠음.
투자일기는 매매기록과 당시 어떤 생각으로 그러한 매매를 했는지 작성
3. 서버에서는 주가, 지수, 뉴스 등 수집
증시에 영향을 미치는 각종 지표들 수집(미국, 일본, 한국 금리, 공포지수, 원자제 가격, 고용지수, CPI 등)
4. 일정 시간이 지난 후 과거의 내 투자일기의 내용과 대조하여 얼마나 맞고 틀린지 분석
(과거 특정일에 대한 분석을 고정적인 미래의 n일이 지난 후 분석 할지, 분석 요청 시점 부터 과거의 모든 이력들에 대하여 분석할지 고민임..)
또한, 투자일기에 대한 맞고 틀리고를 점수화하여 사용자에게 제공.
5. 분석 결과 팝업 하기 전 사용자에게 본인이 생각하기로써
맞고 틀린 것에 대해서 자체적으로 분석하고 평가하여 기록할 수 있는데, 기록하겠냐고 물어봄.
5-1. Yes를 선택하면 AI의 분석내용과 더불어 본인의 분석내용과 LLM을 통한 분석을 통해 본인의 분석이 얼마나 정확도 있게 분석했는지 알려줌
5-2. No 선택하면 AI의 분석내용만 제공
6. 매매일기와 마찬가지로 분석도 날짜 단위로 저장됨. 사용자는 시간이 지나고도 지나간 분석을 재조회할 수 있음.

### 나중에 서비스가 커지면 하고 싶은 것

먼저.... 투자일기를 통해 쌓은 매매기록과 포트폴리오 활용(개인정보 동의받으면 문제없나?)

1. MBTI 처럼 여러 투자 스타일을 미리 분류 해놓고, 포트폴리오 내용에 따라
사용자가 어떤 스타일에 해당되는지 알려줌.
이 또한 매매기록이 변경되면 변경될 수 있기 때문에,
날짜 단위로 저장됨.
(개발 초기에는 포트폴리오로만 분류되게 할 생각이고, 차 후에 내가 어느 정도 이 앱에 대해서
노하우가 쌓이면, 사용자가 투자일기에 적은 생각과 그의 차 후 분석과 매매기록을 통해
사용자의 투자스타일을 정하는 방향으로 업그레이드 하고 싶음...)
2. 어떤 투자 스타일의 투자자가 가장 많은 수익'률'을 내는지 통계하여 순위 제공.(엔터테이너적 요소)
금액으로 순위를 내지 않고 반드시 수익 비율로 내야 함..
3. 어떤 투자 스타일의투자자가 가장 높은 '승률'을 내는지 통계하여 순위 제공.
4. 일별 가장 많은 사람들이 사는 종목(문제가되면 안해야할지도)
5. 일별 가장 많은 사람들이 파는 종목(문제가되면 안해야할지도)
6. 커뮤니티
7. 보유 종목에 관련된 뉴스 피드 기능
8. **투자 교육 콘텐츠 제공**
    - 사용자의 투자 스타일과 분석 결과를 바탕으로 맞춤형 교육 콘텐츠를 추천. 예: 리스크 관리가 약한 사용자에게 "리스크 관리 기초" 강의 추천.
    - **기술**: 콘텐츠 DB + 사용자 데이터 분석 후 추천 알고리즘(협업 필터링).
9. **가상 투자 시뮬레이션**
    - 실제 돈을 투자하기 전에 가상 자산으로 전략을 테스트해볼 수 있는 기능. 사용자가 자신의 투자일기 기반으로 시뮬레이션 가능.
    - **기술**: 가상 계좌 시스템 + 실시간 주가 API 연동.
10. **투자 목표 설정 및 추적**
    - 사용자가 목표(예: "연간 수익률 10%")를 설정하고 진행 상황을 모니터링할 수 있는 대시보드 제공.
    - **기술**: 목표 입력 UI + 진행률 계산 로직 + 시각화.
11. **커뮤니티 내 투자 대회**
    - 가상 자산으로 사용자들이 경쟁하며 투자 실력을 겨루는 이벤트. 상위 랭커에게 보상(예: 프리미엄 기능 무료 이용권) 제공.
    - **기술**: 랭킹 시스템 + 이벤트 관리 모듈.
